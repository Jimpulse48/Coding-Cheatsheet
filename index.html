Coding Cheat Sheet

HTML5

Structure

Boilerplate

A boilerplate code, boilerplate for short, is a section of code that you should add at the start of every project. The boilerplate tells the browser how to interpret the code. The browser can load your website without a boilerplate. Your website, however, may not be as polished as you like.

<!DOCTYPE html>
<html lang='en'>
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<title></title>
<link rel='stylesheet' href='./Resources/css/style.css'>
</head>
<body>
</body>
</html>

Head

The head section contains meta information about your website. Aside from the title, code in the head section will not be viewable to the audience. The contents of the <title> element is shown in the tab section.

Body

The body section contains the bulk of your code. Anything visible to the audience will be in the body section.

Text

Headings

HTML5 has six "levels" of headings:
<h1> is used for main headings
<h2> is used for subheadings
If there are further sections under the subheadings then the <h3> element is used, and so on...

Paragraphs

To create a paragraph, surround the words that make up the paragraph with an opening <p> tag and closing </p> tag.

By default, a browser will show each paragraph on a new line with some space between it and any subsequent paragraphs.

Text Elements

Element Name Description
<b> Bold Words enclosed in this element will appear bold.
<i> Italic Words enclosed in this element will appear italic.
<sup> Superscript Words enclosed in this element will be displayed as a superscript. For example, E=MC2.
<sub> Subscript Words enclosed in this element will be displayed as a subscript. For example, H2O.
<br /> Line Break Adds a line break in the middle of a paragraph.
<hr /> Horizontal Rules Creates a break between themes with a horizontal line.

Semantic Markup - Text

There are some text elements that are not intended to affect the structure of your web pages, but they do add extra information to the pages - they are known as semantic markup.

Element Name Description
<strong> Strong The use of the <strong element indicates that its content has strong importance. Browsers will show the contents of a <strong> element in bold.
Use this element in lieu of the <bold> element.
<em> Emphasis The use of the <em> element indicates emphasis that subtly changes the meaning of a sentance. Browsers will show the contents of a <em> element in italics.
Use this element in lieu of the <i> element.
<blockquote> Blockquote Longer quotes that take up an entire paragraph should be enclosed in this element. Browsers tend to indent the contents of the <blockquote> element.
You should not, however, use this element to indent a piece of text.
<q> Quote Shorter quotes that sit within a paragraph should be enclosed in this element. Browsers should put double quotations are the <q> element.
Many people would avoid using this element.
<abbr> Abbreviations Abbreviations or acronyms should be enclosed in this element. A title attribute on the opening tag is used to specify the full term.

Media

Images

To add an image into a webpage you need to use an <img> element. This is an empty element; there is not a closing tag.

Attribute Name Description
src Source The value of this attribute tells the browser where it can find the image file. This will usually be a relative URL pointing to an image on your own site.
alt Alternate The value of this attribute provides a text description of the image which describes the image if you cannot see it. A browser will show the "alt text" in lieu of a picture.
title Title The value of this attribute provides additional information about the image. Most browsers will display the title in a tooltip when the audience hovers over the image.

Videos

The <video> element has a number of attributes which allow you to control video playback. If the browser does not support the <video> element or the format of video used, it will display whatever is between the opening <video> and close </video> tags.

Attribute Name Description
src Source The value of this attribute tells the browser where it can find the video file. This will usually be a relative URL pointing to an image on your own site.
preload Preload This attribute tells the browser what to do when the page loads. It can have one of three values:
  1. none - The browser should not load the video until the user presses play.
  2. auto - The browser should download the video when the page loads.
  3. metadata - The browser should just collect information such as the size, first frame, track list, and duration.
poster Poster The value of this attribute specifies an image to show while the video is downloading or until the user tells the video to play.
controls Controls When used, this attribute indicates that the browser should supply its own controls for playback.
autoplay Autoplay When used, this attribute specifies that the file should play automatically.
loop Loop When used, this attribute indicates that the video should start playing again once it has ended.

Lists

There are plenty of occasions when we need to use lists. HTML provides us with three different types:

  • Ordered lists are lists where each item in the list is numbered.
  • Unordered lists are lists that begin with a bullet point (rather than characters that indicate order).
  • Definition lists are made up of a set of terms along with the definitions for each of those terms.

List Elements

Element Name Description
<ol> Ordered List Creates an ordered list.
<ul> Unordered List Creats an unordered list.
<li> List Item Every item in a list needs to be placed between an opening <li> tag and a closing </li> tag.
<dl> Definition List Creats a definition list.
<dt> Definition Term This is used to contain the term being defined.
<dd> Definition This is used to contain the definition.

Tables

A table represents information in a grid format. Examples of tables include financial reports, TV schedules, and sports results.

Table Elements

Element Name Description
<table> Table The <table> element is used to create a table. The contents of the table are written out row by row.
<tr> Table Row You indicate the start of each row by using the opening <tr> tag. It is followed by one or more <td> element. At the end of the row you use a closing </tr> tag.
<td> Table Data Each cell of a table is represented using a <td> element.
<th> Table Heading The <th> element is used like the <td> element but its purpose is to represent the heading for either a column or a row.
<thead> Table Head The headings of the table should sit inside the <thead> element.
<tbody> Table Body The body should sit inside the <tbody> element.
<tfoot> Table Footer The footer belongs inside the <tfoot> element.

Table Attributes

There are a few attributes that can affect how the table displays data.

Attribute Name Description
colspan Column Span The colspan attribute can be used on a <th> or <td> element and indicates how many columns that cell should run across.
rowspan Row Span The rowspan attribute can be used on a <th> or <td> element and indicates how many rows that cell should span down the table.
scope Scope The <scope> attribute is used as a semantic to indicate whether a <th> element is a heading for a row or column.

Links

Links are created using the <a> element. Users can click on anything between the opening <a> and close </a> tags. You specify which page you want to link to using the href attribute.

  • Linking to other sites
    • The value of the href attribute should be the full web address for the site, which is known as an absolute URL.
  • Linking to other pages on the same site
    • When you are linking to other pages within the same site, you do not need to specify the domain name in the URL. You can use a shorthand known as a relative URL.

Relative URL

On larger websites it's a good idea to organize your code by placing the pages for each different section of the site into a new folder. Folders on a website are sometimes referred to as directories.

Relative Link Type Definition Example
Same Folder To link to a file in the same folder, just use the file name. (Nothing else is needed.) To link to music reviews from the music homepage: <a href='reviews.html'>Reviews</a>
Child Folder For a child folder, use the name of the child folder, followed by a forward slash, then the file name. To link to music listings from the homepage: <a href='music/listings.html'>Listings</a>
Grandchild Folder Use the name of the child folder, followed by a forward slash, then the name of the grandchild folder,
followed by another forward slash, then the file name.		 To link to DVD reviews from the homepage: <a href='movies/dvd/reviews.html'>Reviews</a>
Parent Folder Use ../ to indicate the folder above the current one, then follow it with the file name. To link to the homepage from the music reviews: <a href=../index.html'>Home</a>
Grandparent Folder Repeat the ../ to indicate that you want to go up two folders (rather than one), then follow it with the file name. To link to the homepage from DVD reviews: <a href='../../index.html'>Home</a>

Forms

The <form> element allows you to collect information from vistors to your site. Whether you are adding a simple search box to your website or you need to create more complicated insurance appliations, HTML forms give you a set of elements to collect data from your users.

Form controls live inside a <form> element. This element should always carry the action attribute and will usually have a method and id attribute too.

action

Every <form> element requires an action attribute. Its value is the URL for the page on the server that wil receive the information in the form when it is submitted.

method

Forms can be sent using one of two methods: get or post.
With the get method, the values from the form are added to the end of the URL specified in the action attribute. The get method is ideal for:

  • short forms (such as search boxes)
  • when you are just retrieving data from the web server (not sending information that should be added to or delted from a database)
With the post method the values are sent in what are known as HTTP headers, As a rule of thumb you should use the post method if your form:
  • allows users to upload a file
  • is very long
  • contains sensitive data (e.g. passwords)
  • adds information to, or deletes information from, a database

Input

The <input> element is used to create several different form controls. The value of the type attribute determines what kind of input they will be creating.
The name attribute identifies the form control and is sent along with the information they enter to the server.

Input type Definition
text Creates a single-line text input
password Creates a single-line text input. The user's input is censored.
radio Creates radio buttons that allow users to pick just one of a number of options.
The value attribute indicates the value that is sent to the server for the selected option.
The checked attribute can be used to indicate which value should be selected when the page loads.
checkbox Creates checkboxes that allow users to select (and deselect) one or more options in answer to a question.
The value attribute indicates the value sent to the server if this checkbox is checked.
The checked attribute can be used to indicate which value should be selected when the page loads.
file Allows the user to upload a file.
submit Creates a submit button.
The value attribute is used to control the text that appears on a button.
image Uses an image in lieu of a submit button. The src, width, height, and alt attributes work just like they would when used with the <img> element.
